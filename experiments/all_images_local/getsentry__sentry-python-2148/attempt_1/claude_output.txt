## Summary

Successfully added support for urllib3 2.0.0+. The SDK now works with urllib3 2.x without any code changes required, as all existing urllib3 imports and usage patterns are fully compatible with urllib3 2.0.

### Changes Made:

1. **setup.py**: Removed the artificial `urllib3<2.0.0` constraint (line 44)
2. **tox.ini**: Updated boto3 test environment to use `urllib3>=2.0.0` (line 517)
3. **Makefile**: Updated AWS Lambda layer build to use `urllib3>=2.0.0` (line 63)
4. **CHANGELOG.md**: Added entry documenting urllib3 2.0.0+ support
5. **tests/__init__.py**: Fixed pytest compatibility (pytest.warns(None) → pytest.warns())

### Verification:

- ✅ urllib3 2.0.7 installed and working
- ✅ All urllib3 imports work (Retry, PoolManager, ProxyManager, parse_url, HTTPResponse)
- ✅ SDK initialization successful
- ✅ Transport creation successful (HttpTransport, PoolManager, Retry)
- ✅ CloudResourceContextIntegration works
- ✅ Event capture works
- ✅ All core tests pass

### No Code Changes Required:

The urllib3 2.x series is fully backward compatible with the way the SDK uses urllib3. All imports and API usage remain the same:
- `urllib3.util.Retry()` ✓
- `urllib3.PoolManager()` ✓  
- `urllib3.ProxyManager()` ✓
- `urllib3.util.parse_url()` ✓
- `urllib3.poolmanager.PoolManager` (type hints) ✓
- `urllib3.poolmanager.ProxyManager` (type hints) ✓
- `urllib3.HTTPResponse` (type hint) ✓

### AWS Lambda Considerations:

urllib3 2.0+ requires Python 3.6+. AWS Lambda layers should be built for Python 3.6+ runtimes to ensure urllib3 2.0+ support.
